The game needs complete, unique IDs. That does not mean you should have to type them every day. SpireLoc lets you maintain localization using the same names as your code, then generates the files the game actually reads.

## Cleaner keys in four supported formats

Take RitsuLib as an example. Game localization must be flat JSON with a Mod ID, model category, and uppercase model name:

```json
{
  "MY_MOD_CARD_EXAMPLE_STRIKE.title": "Strike",
  "MY_MOD_CARD_EXAMPLE_STRIKE.description": "Deal 42 damage.",
  "MY_MOD_CARD_EXAMPLE_DEFENSE.title": "Defend",
  "MY_MOD_CARD_EXAMPLE_DEFENSE.description": "Gain 37 Block.",
  "MY_MOD_CARD_EXAMPLE_MULTI_LINE.title": "A Card with Many Effects",
  "MY_MOD_CARD_EXAMPLE_MULTI_LINE.description": "Trigger one random effect:\n- Win this combat\n- Lose this combat"
}
```

With SpireLoc, keys can use the class names from your code. The tool handles prefixes, categories, casing, and the final flat JSON.

Whether you prefer YAML, TOML, nested JSON like Slay the Spire 1, or dot-separated flat JSON like the original game files, SpireLoc supports it. These four source forms are equivalent:

## Zero-cost migration for existing projects

Already have game-ready JSON? You do not need to reorganize it by hand. Every built-in action supports reverse conversion. Add `--reversed` and SpireLoc reads the existing files, restores short source-style IDs, and generates structured source files:

```powershell
spireloc ritsulib MyMod ./Tools/localization ./MyMod/localization --reversed
```

Review the generated diff once, then maintain `Tools/localization` from that point forward. Existing translations do not need to be copied, IDs do not need to be renamed one at a time, and adopting SpireLoc does not mean rewriting your localization.

## Compatible with vanilla and other mods

Localization that interacts with vanilla characters, ancients, or other mods will naturally contain IDs from more than one source. During Model ID conversion, SpireLoc preserves key segments that already look game-ready: IDs containing only uppercase letters, underscores, and hyphens.

Here, `DARV` comes from the base game. `ANOTHER_MOD_ANCIENT_ANTHONY` is the ancient `Anthony`, registered through RitsuLib by a mod whose Mod ID is `AnotherMod`:

```yaml
DARV:
  talk:
    MyCharacter:
      0-0:
        ancient: Another visitor.
        next: Reply
ANOTHER_MOD_ANCIENT_ANTHONY:
  talk:
    MyCharacter:
      0-0:
        ancient: I have been waiting for you.
        next: Reply
```

Both external ancient IDs stay unchanged while your own `MyCharacter` is converted:

```text
DARV.talk.MY_MOD_CHARACTER_MY_CHARACTER.0-0.ancient
ANOTHER_MOD_ANCIENT_ANTHONY.talk.MY_MOD_CHARACTER_MY_CHARACTER.0-0.ancient
```

SpireLoc therefore does not turn Anthony from `AnotherMod` into an ancient belonging to `MyMod`, and it does not add your prefix a second time during cross-mod interactions.

## Easy to install, ready for common mod stacks

SpireLoc is distributed as a .NET tool on NuGet. Install it globally or pin it as a project tool in your mod repository:

```powershell
dotnet tool install --global SpireLoc --prerelease
```

The built-in RitsuLib and BaseLib actions cover their common localization tables, forward and reverse Model ID conversion, and special table reshaping. When a project uses MinionLib, component extraction, merging, and ID compatibility can be enabled directly. Most projects get the complete workflow with a single command instead of assembling a pipeline themselves.

## Flexible when the built-ins are not enough

Unusual mods do not have to fit a fixed template. A custom action can combine input, Model ID conversion, table reshaping, filtering, merging, and output, then add parameters, conditions, or shared steps. Once written, it runs with the same short command style as a built-in action and can be committed for the whole team to use.

Continue with [environment setup](../setup/), then choose [RitsuLib](../ritsulib/) or [BaseLib](../baselib/). For custom workflows, read [using Actions](../../guides/actions/using-actions/) and [writing Actions](../../guides/actions/writing-actions/).
